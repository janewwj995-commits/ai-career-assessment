# AI职业风险测试项目 - 工作规则

## 📋 项目基本信息
- 项目名称：AI时代职业结构风险自测
- 品牌名称：AI共学实验室（AI Co-Learning Lab）
- 核心目标：搭建一个网页版的AI职业风险测试问卷系统

---

## 🎯 项目交付要求

### 技术栈选择
- **必须使用**：纯 HTML + CSS + JavaScript（对小白最友好，易于理解和维护）
- **禁止使用**：复杂的框架（React、Vue、Angular等）、小众技术
- **必须兼容**：现代浏览器（Chrome、Safari、Firefox、Edge）

### 代码质量标准
1. **清晰注释**：所有代码必须有中文注释，解释关键逻辑
2. **命名规范**：文件和变量命名要通俗易懂，使用有意义的名称
3. **结构清晰**：代码结构要模块化，便于后续修改
4. **测试保证**：每个功能完成后必须经过测试，确保无明显bug

### 功能要求
1. **问卷系统**：完整实现20道题目的交互
2. **评分逻辑**：严格按照文档实现A/B/C三轴评分和分型判断
3. **报告生成**：生成4种类型的个性化报告
4. **响应式设计**：必须在电脑和手机上都能正常显示和使用
5. **用户体验**：界面简洁美观，操作流畅

---

## 🗣️ 沟通规则

### 提问方式
- **规划阶段**：可以一次性提出多个相关问题
- **开发阶段**：建议每次聚焦一个具体功能模块
- **任何时候**：有疑问随时提出，不要犹豫

### 沟通语言
- **必须使用**：通俗易懂的中文
- **避免使用**：过多专业术语，必要时要解释清楚
- **解释方式**：用比喻和例子来说明技术问题

### 反馈机制
- **每个模块完成后**：主动展示成果，等待用户确认
- **有多个方案时**：先说明各方案的优缺点，让用户选择
- **遇到问题时**：及时沟通，不要私自做重大决定

---

## 📝 开发流程规则

### 阶段划分
1. **阶段一**：搭建基础网站结构和UI框架
2. **阶段二**：实现问卷界面和答题逻辑
3. **阶段三**：实现评分算法和分型判断
4. **阶段四**：实现报告生成和展示
5. **阶段五**：优化和测试，部署上线

### 每个阶段的要求
- **明确目标**：开始前明确本阶段要完成什么
- **小步快跑**：每个阶段拆分成小任务，逐个完成
- **及时确认**：完成后立即展示给用户确认
- **记录文档**：重要决策和修改要记录下来

---

## ⚠️ 重要注意事项

### 必须遵守
1. **严格按照文档**：问卷内容、评分标准、报告模板必须与Word文档完全一致
2. **数据匿名化**：如果涉及数据存储，必须确保用户隐私安全
3. **不做过度承诺**：网站上的文案要符合策划书中的风险控制要求
4. **保持简洁**：不要添加不必要的功能，聚焦核心需求

### 禁止行为
1. 不要私自改变需求或功能
2. 不要使用用户不理解的技术
3. 不要一次性写太多代码而不测试
4. 不要跳过用户确认就进入下一阶段

---

- [ ] 报告生成正常，4种类型都能正确展示
- [ ] 界面美观，符合品牌调性
- [ ] 没有明显的bug或错误

---

## 📞 紧急情况处理

如果遇到以下情况，立即暂停并沟通：
1. 发现需求理解有重大偏差
2. 遇到技术障碍无法在短时间内解决
3. 需要对原计划做重大调整
4. 用户反馈与预期不符
